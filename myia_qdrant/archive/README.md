# Archive - myia_qdrant

Ce répertoire contient les fichiers archivés du projet myia_qdrant, organisés pour préserver l'historique sans encombrer la structure principale.

## 📁 Structure

```
archive/
├── reports/                    # Rapports historiques de maintenance
│   ├── CLEANUP_REPORT_20251013.md
│   └── CONSOLIDATION_REPORT_20251013.md
│
└── diagnostics/                # Fichiers de diagnostic archivés
    ├── scripts/                # Scripts de diagnostic temporaires
    ├── logs/                   # Logs et traces de diagnostic
    └── *.md, *.json            # Rapports et données de diagnostic
```

## 📊 Contenu

### reports/
Contient les rapports de maintenance et de consolidation datés, documentant les opérations importantes effectuées sur le projet.

- **CLEANUP_REPORT_20251013.md** : Rapport de nettoyage du 13 octobre 2025
- **CONSOLIDATION_REPORT_20251013.md** : Rapport de consolidation du 13 octobre 2025

### diagnostics/
Contient tous les fichiers liés aux diagnostics et incidents passés, organisés par type :

#### diagnostics/scripts/
Scripts PowerShell utilisés pour diagnostiquer des problèmes spécifiques :
- Scripts d'analyse de freeze
- Scripts de vérification de correction
- Scripts de diagnostic urgent
- Scripts de recréation de collection

#### diagnostics/logs/
Fichiers de logs (.txt, .log) générés lors des diagnostics :
- Logs de freeze
- Logs post-correction
- Logs complets d'analyse

#### diagnostics/ (racine)
Rapports et données de diagnostic (.md, .json) :
- Rapports d'analyse de cause racine
- Rapports de résolution finale
- Rapports d'incident post-correction
- Fichiers de configuration de backup
- Scans de collections

## 🎯 Pourquoi archiver ?

Ces fichiers sont **préservés** plutôt que supprimés pour :

1. **Traçabilité** : Maintenir un historique complet des opérations
2. **Documentation** : Référence pour incidents similaires futurs
3. **Audit** : Preuve des actions prises et des résultats obtenus
4. **Apprentissage** : Base de connaissance pour éviter les erreurs passées

## ⚠️ Note importante

Ces fichiers sont **en lecture seule** et ne doivent **pas être modifiés**. Ils servent uniquement de référence historique.

Si vous avez besoin de consulter ces fichiers, vous pouvez les ouvrir directement depuis ce répertoire. Si vous devez effectuer de nouvelles opérations de diagnostic, créez de nouveaux fichiers dans les emplacements appropriés de la structure principale.
